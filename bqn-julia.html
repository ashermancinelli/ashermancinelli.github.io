Leetcode problem: Maximum Gap.

As C++ is my primary language, thatâ€™s where I started with this problem.

This was my first pass to see if I understood the problem well enough:

int maximum_gap(vector<int>& nums) {
  if (nums.size() < 2)
    return 0;
  std::sort(nums.begin(), nums.end());
  int m=0;
  for (int i=1; i < nums.size(); i++) {
    m = std::max(m, std::abs(nums[i]-nums[i-1]));
  }
  return m;
}

This cleaned up version that uses Range V3 (godbolt link), however it may not use linear time and linear extra space. If you want to be a stickler about the time and space requirements, you can refer to Razâ€™s final Julia solution or my previous C++ solution.

int solve(auto& v) {
    if (v.size() < 2) return 0;
    sort(v);
    return max(views::transform(v, v | views::drop(1),
        [] (auto a, auto b) {
            return std::abs(a-b);
        }));
}

This uses a suggestion in an issue in the Range V3 github. I would rather have an adjacent difference view I could pipe to.

function solve(nums)
  if length(nums) < 2
    return 0

  nums = sort(nums)

  n = nothing
  for i in nums
    if n == nothing
      n = i

    m = max(m, abs(n-i))
    n = i
  end

function sol(arr)
    # Bail early if too short
    length(arr) < 2 && return 0
    # take the maximum of the diff
    mini = minimum(arr)
    # Use in-place RadixSort to modify `arr`
    sort!(arr, alg = RadixSort)
    # Take the max of the diff
    maximum(diff(arr))
end
